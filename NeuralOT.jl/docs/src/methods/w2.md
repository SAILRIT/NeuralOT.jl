# W₂ Monge map via ICNNs (Makkuva et al., 2020)

## Problem

For squared-Euclidean cost, Brenier's theorem states that the unique
optimal Monge map between μ and ν (assuming μ absolutely continuous) is
the gradient of a convex function:

```math
T^\star(x) = \nabla \varphi^\star(x).
```

Makkuva, Taghvaei, Oh, and Lee (2020) propose parameterising this convex
function as an **Input Convex Neural Network** (ICNN; Amos et al., 2017)
and solving the saddle-point dual

```math
\inf_{f \in \mathrm{cvx}} \sup_{g \in \mathrm{cvx}}
  \mathbb{E}_\mu[\langle X, \nabla g(X) \rangle - f(\nabla g(X))]
  + \mathbb{E}_\nu[f(Y)].
```

Both `f` and `g` are ICNNs; their gradients are valid transport maps, and
at optimum `∇g` is the W₂ Monge map from μ to ν.

## ICNNs

An [`ICNN`](@ref) is a feed-forward network that is convex in its input by
construction. We enforce convexity by applying `softplus` to the hidden-
to-hidden weights during the forward pass (so they are strictly positive,
satisfying the sufficient condition in Amos et al.) and by using
`softplus` as the activation (convex and non-decreasing).

```julia
f = ICNN(2, [64, 64, 1])        # scalar convex function on R²
x = randn(Float32, 2, 16)
vals  = f(x)                     # (1, 16)
grads = NeuralOT.grad_x(f, x)    # (2, 16) — the Brenier map candidate
```

## Usage

```julia
result = solve_w2(sample_μ, sample_ν;
                  dim=2, widths=[64, 64, 1],
                  steps=3_000, inner_steps=10,
                  batch=256, lr=1e-4, seed=0)

X = sample_μ(500)
T_X = monge_map(result, X)       # applies ∇g
```

## Tuning notes

- **widths**: the last entry must be 1. Start small (e.g. `[32, 32, 1]`)
  and increase only if under-fitting.
- **inner_steps**: more inner iterations give a better inner sup
  approximation but slow training. 5–15 is typical.
- **lr**: saddle-point training is sensitive; if oscillating, halve `lr`.
- **Validation**: check [`sinkhorn_divergence`](@ref) between
  `monge_map(result, X)` and a fresh draw from ν. On 2D Gaussians it
  should drop below 0.01.

## Comparison to `flow_match`

| Aspect | `solve_w2` | `flow_match` |
|---|---|---|
| Output | Deterministic map `T(x)` | ODE flow `x(t)` |
| Cost | Fixed: squared Euclidean | Implicit via interpolation |
| Theory | Brenier-valid map | Not guaranteed optimal |
| Training stability | Saddle-point (harder) | Regression (easier) |
| Scalability | Moderate | Very high |
